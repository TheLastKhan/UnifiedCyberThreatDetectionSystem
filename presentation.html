BERT (DistilBERT) - Deep Contextual Understanding

Technical Details

  • Architecture: DistilBERT (66M parameters)
  • Training Data: 31,000+ labeled emails
  • Accuracy: 94-97%
  • Inference Time: ~45ms
  • Model Size: 250MB
  • Tokenizer: WordPiece (30,522 vocab)

Why BERT?

  • Understands context: "Click here to verify" is suspicious
  • Handles typos: "Paypa1" vs "PayPal"
  • Captures semantic meaning: urgency, threats
  • Transfer learning from pre-trained model

Example Analysis

POST /api/email/analyze/bert

{"subject": "URGENT: Account suspended!",
 "sender": "security@paypal-fake.com",
 "body": "Your account has been compromised..."}

Result: PHISHING

Confidence: 97%

Risk Level: CRITICAL

Processing: 45ms

FastText - Ultra-Fast Processing

Technical Details

  • Developer: Facebook AI Research
  • Architecture: Word embeddings + Linear classifier
  • Accuracy: 90-94%
  • Inference Time: <1ms (sub-millisecond)
  • Model Size: 881MB (includes word vectors)
  • Uses subword information (character n-grams)

Why FastText?

  • Extremely fast: millions of emails per hour
  • Handles out-of-vocabulary words
  • No GPU required for inference
  • Best for real-time high-volume processing

Speed Comparison

FastText <1ms
TF-IDF + RF ~25ms
BERT ~45ms

Use Case

Real-time email gateway filtering where latency is critical. Can process 1M+ emails/hour on single CPU.

TF-IDF + Random Forest - Explainable Baseline

Technical Details

  • Vectorization: TF-IDF (Term Frequency-Inverse Document Frequency)
  • Classifier: Random Forest with 100 trees
  • Balancing: SMOTE oversampling
  • Accuracy: 89.75%
  • ROC-AUC: 97.50%
  • Model Size: 50MB

Why TF-IDF + RF?

  • Fully explainable with LIME
  • Shows which words triggered detection
  • Reliable baseline without deep learning
  • Fast training and retraining

LIME Explanation Example

urgent+0.35
verify+0.28
click_here+0.22
regards-0.05

Result: PHISHING (89% confidence)

Hybrid Ensemble Approach

Final_Score = (BERT x 0.5) + (FastText x 0.3) + (TF-IDF x 0.2)

Explainable AI with LIME

Why Explainability Matters

  • Builds trust in AI-driven security decisions
  • Helps analysts learn from model reasoning
  • Enables verification of correct detection
  • Reduces false positive investigation time
  • Meets regulatory compliance requirements

How LIME Works

  • Perturbs input around the prediction
  • Trains local interpretable surrogate model
  • Calculates feature importance weights
  • Returns human-readable explanations
