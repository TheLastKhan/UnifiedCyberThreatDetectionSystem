Problem Statement & Motivation

Core Challenges

1. Siloed Security Infrastructure

Email gateways and Web Application Firewalls (WAF) operate in isolation. Cross-vector attacks (Email -> Web) are often missed because no single system sees the full picture.

2. The "Black Box" Trust Issue

Advanced Deep Learning models (LSTM, CNN) offer high accuracy but zero transparency. Security analysts cannot trust "magic" decisions without knowing why an alert was triggered.

3. Reactive vs. Proactive

Traditional systems rely on signatures of known attacks. They fail against zero-day exploits or subtle behavioral anomalies.

Impact & Motivation

  • Surging Phishing Attacks: 202% increase in phishing attacks in H2 2024 (SlashNext Report).
  • Dwell Time: The time between compromise and detection remains critical. Disconnected tools increase this delay.
  • Alert Fatigue: Analysts are overwhelmed by false positives. Without context (XAI), investigating every alert is impossible.

Project Objectives:

  • Develop a Unified Platform correlating Email & Web data.
  • Implement Explainable AI (LIME/SHAP) for transparency.
  • Achieve >90% Accuracy in phishing detection.

Literature Review & Gap Analysis

Study Technique Focus & Results Gap / Limitation
Bergholz et al. (2010) Heuristics / Rules Early phishing detection. Analysis of headers & content. High false positive rate. Fails against new/zero-day attacks.
Hiransha et al. (2018) Deep Learning (LSTM) Sequential email text analysis. High accuracy (94%). "Black Box" model. No explainability provided to analysts.
Zhang et al. (2019) Isolation Forest Web application firewall rule generation. Isolated Scope. Only analyzes web logs, ignores email context.
Ghafir et al. (2018) Hidden Markov Models APT detection via correlation. Focused on network layer only; missed application layer/email vectors.
This Project (2025) RF + IF + LIME Multi-Vector Correlation. Addresses integration gap + Adds XAI layer.

Methodology: Email Phishing Detection

1. Feature Engineering (5,015 Features)

We extract a rich set of features to capture both semantic and structural properties:

  • Content-Based: 5000 TF-IDF (Term Frequency-Inverse Document Frequency) features (N-grams).
  • Keyword Features: Counts of "Urgent" words (e.g., "Immediate", "Expire"), Financial terms ("Bank", "Verify").
  • Structural Features: Capital letter ratio, HTML tag density, Punctuation counts.
  • URL Features: IP in URL, Suspicious TLDs (.tk, .ml), URL shorteners.

2. Algorithm: Random Forest

  • Ensemble Learning: Uses 100 Decision Trees (n_estimators=100).
  • Why RF? Robust against overfitting on high-dimensional text data. Provides feature importance natively.

3. Explainability (LIME)

Local Interpretable Model-agnostic Explanations:

  • Perturbs the input text (removes words) to see how prediction changes.
  • Output: "The word 'Account' contributed +0.15 to Phishing score."
  • Provides actionable insights to the analyst.

Methodology: Web Anomaly Detection

1. Behavioral Feature Extraction

We aggregate raw logs by IP address to create a behavioral profile:

  • Volume Metrics: Request count, Request rate (req/min).
  • Error Metrics: Ratio of 4xx (Client Error) and 5xx (Server Error) status codes.
  • Temporal Metrics: Variance in time between requests (detects automated bots).
  • Diversity: Number of unique User-Agents, Paths accessed.

2. Algorithm: Isolation Forest

  • Unsupervised Learning: Does not require labeled "attack" data.
  • Logic: Anomalies are rare and distinct. They are "isolated" earlier in the random decision trees.
  • Scoring: Returns an anomaly score. Negative values indicate high anomaly.

3. Signature Matching

Layered defense using Regex for known patterns:

  • SQL Injection: UNION SELECT, OR 1=1.
  • XSS: <script>, javascript: alerts.

Unified Correlation Logic

Risk Scoring Formula

Score = min(100, (EmailRisk * 0.4) + (WebRisk * 0.4) + CorrelationBonus)

This weighted formula ensures that a high risk in one vector alerts the system, but a correlation between vectors pushes the score to CRITICAL.

Components:

  • Email Risk (0-40): Based on RF Probability.
  • Web Risk (0-40): Based on Anomaly Score & Patterns.
  • Correlation Bonus (0-20): Added only if rules trigger.

Correlation Rules

1. IP Coincidence Rule

Logic: Triggers if the Sender IP (from email headers) matches a Source IP in Web Logs targeting the server.

Impact: Strong indicator of a targeted campaign.

2. Temporal Proximity Rule

Logic: Triggers if a High-Risk Email and a Web Anomaly occur within a 1-hour window.

3. Pattern Similarity

Logic: Detects if Email Subject ("Password Reset") aligns with Web Target Path (/admin/login brute force).
